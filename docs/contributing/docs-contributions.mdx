## Edit on GitHub

1. open the page,
2. click edit,
3. submit a pull request.

## Local docs workflow

1. clone repository,
2. install Mintlify CLI (`npm i -g mint`),
3. run `mint dev` in `docs/` and verify the language switcher for English and Español,
4. validate links with `mint broken-links`,
5. open pull request.

## Docs i18n playground workflow

Use the docs i18n config at `docs/i18n.docs.jsonc`.

### Prerequisites

- install hyperlocalise from the repo root (`go run . --help`),
- set provider credentials before non-dry-run generation (for example `OPENAI_API_KEY` when using the default OpenAI profile),
- run all commands from `docs/`.

### Reproducible validation sequence

```bash
# 1) plan translation work without writing files
go run .. run --config i18n.docs.jsonc --dry-run

# 2) generate target locale docs
go run .. run --config i18n.docs.jsonc

# 3) verify links are valid after generation
mint broken-links
```


## Writing standards

- use active voice and second person,
- keep examples executable,
- lead each page with goal and prerequisites,
- keep command flags aligned with current CLI help output.
